javascript基础之面向对象

Sep 30, 2017 | javascript

最近在看《你不知道的JavaScript》一书，发现其中有些部分的讲解是自己之前未曾想到过的东西，故借此机会重新梳理JavaScript高级部分的内容和知识，对自己的知识点回炉重造！

JavaScript中最重要的内容包括面向对象，原型，原型链，继承，this和闭包，事件等等，而对象则是所有内容的基础。关于JavaScript中的对象和其他面向对象语言有所区别！

对象

javascript中大部分内容都是对象！

​ 注意：此处没有说一切，为什么呢？

​ 原因如下：

​ 对于JavaScript来说，简单基本类型（string、boolean、number、null、undefined）并不是对象。typeof null 之所以返回“object”原因是在底层判断中，二进制前三位为0则被判定为object，而null全为0[来自你不知道的js]。

​ ECMA-262给对象的定义是：无序属性的集合，其属性可以包含基本只，对象或者属性

​ 即：对象是键值对的集合，对象是由属性和方法构成的。

在js中包含的几种对象：

  • 原生对象
    +内置对象
    +宿主对象

原生对象

​ ECMA-262定义的类（引用类型），在运行过程中动态创建的的对象，需要使用new进行实例化才能使用。包含内置对象，也就是说实际上是一些内置的构造函数。

包含：Object、Function、Array、String、Boolean、Number、Date、RegExp、Error、EvalError、RangeError、ReferenceError、SyntaxError、TypeError、URIError、ActiveXObject(服务器方面)、Enumerator(集合遍历类)、RegExp（正则表达式）

内置对象

​ 内置对象是原生对象的子集，在引擎初始化时就已经被创建的对象，使用时不需要实例化

​ Global（全局对象） 和Math

​ 全局对象是预定义的对象，作为JavaScript的全局函数和全局属性的占位符。全局对象既没有构造函数，也无法实例化一个新的全局对象。在js中，全局对象就是window对象，表示允许JavaScript的web浏览器窗口

1>:获取属性

​ []语法相对使用更为广泛，关键字也可以用，数字也可以用

复制对象

​ 浅拷贝：简单的拷贝对象，没有改变对象内容的指向，实际上拷贝对象和原对象在内存中指向同一个地址；

​ 深拷贝：对原对象的每个属性进行递归复制，重新创建一个新的对象，新对象在内存中与原对象的地址不相同；

​ 为什么会有浅拷贝和深拷贝的区别？原因是在js中，基本类型的值都是按值传递，而对象则是按引用传值，所有在复制对象时，要考虑到实际的指向，而不能简单的复制对象的指针。

对象原型

​ JavaScript也被称之为“基于原型的对象”。其原因是每个对象都有一个原型对象。根据原型对象（相当于模板）继承属性和方法。

​ 在传统的 OOP 中，首先定义“类”，此后创建对象实例时，类中定义的所有属性和方法都被复制到实例中。在 JavaScript 中并不如此复制——而是在对象实例和它的构造器之间建立一个链接（它是proto属性，是从构造函数的prototype属性派生的），之后通过上溯原型链，在构造器中找到这些属性和方法。

没有官方的方法用于直接访问一个对象的原型对象——原型链中的“连接”被定义在一个内部属性中，在 JavaScript 语言标准中用 [[prototype]] 表示（参见 ECMAScript）。然而，大多数现代浏览器还是提供了一个名为 __proto__ （前后各有2个下划线）的属性，其包含了对象的原型。

prototype属性

​ 该属性值是一个对象，存放继承成员的属性和方法。

​ Object.prototype

constructor属性

​ 每一个实例对象都有，指向该实例的构造器函数

何为继承？

在数学几何中，有这样一种现象，三角形，矩形，六边形，八边形等统称为多边形，而正方形是一种边长度相等的特殊矩形，故而，正方形–矩形–多边形-几何图形就构成一种继承关系。

我们常说：JavaScript是一种基于原型的语言。而原型与原型之间通过继承关系形成原型链。
