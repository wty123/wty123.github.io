javascript基础之事件

Jun 13, 2018 | javascript

​ W3C如是说：事件是可以被js侦测到的行为

​ 事件通常与函数配合，事件触发浏览器的动作，驱动函数执行。

​ 触发某个事件，浏览器会产生相应的行为，在这个触发过程中，事件流会经过整个DOM树，从document到触发事件的DOM节点之间，子元素和父元素都会收到该事件，都收到该事件，那么该如何处理呢，微软和网景给出了两种截然相反的方式，事件冒泡和事件捕获。

​

事件冒泡与事件捕获：

​ 举例： 鼠标放到li上对应的li背景变灰

两种写法，一个在事件冒泡阶段进行处理，一个在事件捕获阶段进行处理；

事件绑定：

​ 举例： 假如有一个div，点击div的过程中，先点击document，事件传到div，div有子元素时继续往下传，最后又冒泡式的传递回document;

  • 第一种 【普通事件绑定】 同一个dom节点只能注册一个事件，后续注册会覆盖

  • 第二种 【标准事件绑定】同一个dom元素注册多个同种事件

    target.addEventListener(type, listener[, useCapture]);

    type：事件名

    listener:处理函数

    useCapture:可选，Boolean,默认为false，表示事件传播方式为事件冒泡；true,表示事件传播方式为事件捕获；

  • ie浏览器处理方式

​ 阻止事件的传播：

​ • 在W3c中，使用stopPropagation（）方法

​ • 在IE下设置cancelBubble = true；

​ 阻止事件的默认行为：

​ • 在W3c中，使用preventDefault（）方法；

​ • 在IE下设置window.event.returnValue = false;

事件委托：

​ 给父元素绑定事件，用来监听并找到哪个子元素的冒泡事件；

​ 优点：减少DOM操作的次数，减少浏览器的重绘和重排，提供代码性能

​ 事件委托是在js的虚拟内存中完成的

​ 事件委托利用事件冒泡，只指定一个事件处理程序，就可以管理某一类的所有事件。例如，可以在DOM树中尽可能高的层次添加一个事件处理程序，之后就可以操作事件目标元素：

关于DOM的级别：

DOM1：

19998年成为W3C的标准，有两个模块组成，DOM核心和DOM HTML。

DOM核心规定如何映射基于XML的文档结构，以便于简化文档中任意部分的访问和操作。DOM HTML在核心基础上扩展，添加了对HTML的对象和方法

DOM2:

在原有基础上扩充了鼠标和用户界面事件，遍历，通过对象接口增加了对css的支持
