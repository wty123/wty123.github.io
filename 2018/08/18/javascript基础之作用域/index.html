javascript基础之作用域

Aug 18, 2018 | javascript

对于JavaScript，我们一直说它是“解释执行”的语言，而在《你不知道的JavaScript》一书中，作者认为JavaScript也是编译语言，而此处所谓的编译语言，并不像其他语言那样，提前编译。

传统编译语言：

​ 1.词法分析：将字符串分解为代码块，称之为代码单元；

​ 2.语法分析：将上一步结果转换为元素组成的结构树；

​ 3.代码生成：将上一步的结构树转换为机器语言。

而JavaScript

编译过程远没有这么简单，由于编译并不是在构建之前，故没有大量时间进行优化，此外JavaScript的编译大部分发生在代码执行前几微秒。

var a = 2;

在JavaScript编译器看来，分为两步，声明和赋值： 看到声明，编译器会在作用域中查找是否有该名称的变量，有则继续赋值，无则需要先声明该变量。赋值操作会在引擎中操作，引擎同样在作用域中查找是否存在该变量，有则赋值，无则抛出异常。

什么是作用域

作用是被执行（被声明）变量所在的上下文；

作用域是编译器声明、引擎确认的变量关系集合；

作用域是一套规则，用于确定引擎在何处查询变量；

词法作用域

顾名思义，就是定义在词法分析阶段的作用域，它是由书写代码时函数的位置来决定的

eval()函数，传入一个参数，该函数会把这个参数插入到代码中，并遮蔽了外部作用域中的同名变量，在严格模式下无效；

尽量不要使用eval和with，它会影响当前词法作用域。

函数作用域

属于该函数的全部变量，都可以在整个函数的范围内使用及复用，倘若没有暴露在外部作用域，那么外部作用域则无法访问到。

对于函数而言，定义函数的方式分为三种：函数表达式，函数声明，new Function（）

区分函数声明和函数表达式最简单的方法是看function关键字出现在声明中的位置，如果function是声明中的第一个词，那么就是一个函数声明，否则为函数表达式。

据此方法，自定义函数也是函数表达式，并不是函数声明。

这句话很容易理解吧？

对于函数声明而言，定义一个函数，最开始以function 开头，而写一个函数表达式的时候，最开始是以一个变量开头，相当于将函数赋值给这个变量，故而此处也就可以理解为什么函数声明可以提前，而函数表达式不能提前

块作用域

JavaScript非其他语言一般有块作用域

何为块级作用域？

在其他类C 的语言中，由花括号包裹起来的就是一个块级作用域。而在js中，只有全局作用域和函数作用域，也就是全局变量和局部变量的概念。不过可以使用立即执行函数模拟块级作用域。在立即执行函数（IIFE）中，所有的变量都是局部变量，并不会污染到全局。

在ES6中，使用let替换var

ES6明确规定，如果区块中存在let和const命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域。凡是在声明之前就使用这些变量，就会报错。

闭包

概念性理解，闭包是指有权访问另一个函数作用域中的变量的函数。

实质上闭包是我们在写代码时基于词法作用域的产物，当函数能够记住并访问其所在的词法作用域时，闭包已经悄无声息的出现在我们面前

类似这样的效果就是一个简单的闭包

本质上对于其他的应用场景，都有闭包的身影存在，定时器、Ajax请求、事件管理器、异步等，所使用的回调函数就是在使用闭包。

上述的方法在ES5中只能这样写，如果在ES6中，直接使用let声明一个变量j就可以搞定！

闭包大多在源码中使用较多，在模块以及封装过程中，闭包能够找到自己的舞台，大放异彩。
